# ADR-008 — v0 Nucleus API Monolith
**Status:** ACCEPTED · 2026-07-08 · Chief Architect (technical authority per working protocol)

## Decision
GateWay Dispatch v0 surfaces (driver API/UI, command board, order form) are hosted INSIDE the deployed events service on the existing Railway service, as a single "Nucleus API" app.

## Why
Railway root-dir is pinned to services/events; adding a second service requires founder taps and doubles cost. One service, one deploy, zero-tap shipping via git push. Velocity per SD-11/SD-14.

## Consequences
- app/dispatch.py + app/ui/ live in services/events temporarily. Impurity is RECORDED, not hidden (SD-16).
- Every dispatch action writes the OWNED Postgres event log (authoritative) and mirrors to Airtable events (founder visibility). The truth now lives on Fivestone infrastructure.
- Split into services/dispatch at M3 when Identity/Access land.
